# История проекта

## Обработка 1С (ОбменССайтом)

### Структура обработки
- Реквизиты: АдресСервера, КлючAPI
- Табличные части: СопоставлениеПолей, СопоставлениеЦен, ТоварыДляВыгрузки, ТоварыССайта, ШаблонРеквизитов
- Реквизит формы: Лог (тип Строка, не ТекстовыйДокумент — был баг с НачатьРедактирование)
- Код модуля формы: 1c/МодульФормы.bsl

### Ключевые изменения

**Исправлен баг с логом**
- Лог был типа ТекстовыйДокумент, использовал НачатьРедактирование/ЗакончитьРедактирование — это клиентские методы, на сервере не работают
- Заменён на тип Строка, конкатенация через Символы.ПС

**Добавлена поддержка дополнительных реквизитов номенклатуры**
- Процедура ЗаполнитьСписокРеквизитовНоменклатуры — добавлен перебор ПланыВидовХарактеристик.ДополнительныеРеквизитыИСведения.Выбрать(), доп. реквизиты добавляются с префиксом "ДопРеквизит." + УИД и меткой "[Доп] Наименование"
- Выгрузка на сайт — если реквизит начинается с "ДопРеквизит.", ищем значение в табличной части Ном.ДополнительныеРеквизиты по свойству
- Загрузка с сайта (создание номенклатуры) — если сопоставленный реквизит доп., пишем в табличную часть ДополнительныеРеквизиты
- **Попытка 1:** Использовали ПВХ.ДополнительныеРеквизитыИСведения.Выбрать() в цикле "Для Каждого" — ошибка "Итератор для значения не определен"
- **Попытка 2:** Заменили на SQL-запрос к ПланВидовХарактеристик.ДополнительныеРеквизитыИСведения с фильтром НЕ ЭтоДополнительноеСведение — РАБОТАЕТ

**Добавлена Группа (Родитель) в список реквизитов обработки**
- Родитель — стандартный реквизит справочника, не попадает через МетаданныеНоменклатуры.Реквизиты
- Добавлен вручную в СтандартныеРеквизиты с представлением "Группа"
- При выгрузке Ном.Родитель вернёт ссылку на группу (папку) номенклатуры

**Добавлено поле Группа (product_group)**
- БД: миграция V0029 — колонка product_group VARCHAR(200) в products
- Backend catalog-products: GET/POST/PUT + поиск
- Backend 1c-exchange: schema/import/export
- Frontend Catalog.tsx: форма + карточка просмотра
- 1С schema: поле product_group с label "Группа"
