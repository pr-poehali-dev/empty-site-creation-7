# Правила работы

## Главное правило — СНАЧАЛА ПЛАН

1. Получил задачу от пользователя
2. Расписал подробный план по пунктам
3. Дождался явного подтверждения ("да", "давай", "вперёд", "согласен")
4. Только после подтверждения приступил к выполнению

**Без подтверждения плана — НЕ ПИСАТЬ КОД. НЕТ ИСКЛЮЧЕНИЙ.**

## Термины проекта

### Категории пользователей
- **Владелец** — определяется по номеру телефона, один (пока). Имеет полный доступ
- **Управленцы** — имеют доступ в админ-панель, интерфейс зависит от роли
- **Обычные пользователи** — отдельный модуль авторизации (будет позже)

### Роли управленцев
- Управляющий
- Менеджер опта
- Менеджер розницы
- Продавец

### Статусы управленцев
- `not_authorized` — владелец добавил номер, человек ещё не привязал Telegram
- `pending` — привязал Telegram, владелец получил уведомление в Telegram, ждёт авторизации
- `authorized` — владелец заполнил данные (Имя, Фамилия, Роль) и авторизовал

## Незавершённые задачи

1. Удаление управленца — полное удаление из списка неавторизованных
2. Доработка /admin для управленцев — authorized управленец должен попадать на свой dashboard, а не на dashboard владельца
3. Dashboard управленца — отдельная страница с интерфейсом в зависимости от роли
4. Редактирование управленца — изменение имени, фамилии или роли уже авторизованного
