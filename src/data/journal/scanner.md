### Контекст задачи
Подключаем Bluetooth-сканер штрихкодов к веб-приложению на iPhone. Проблема iOS Safari: при подключённой внешней BT-клавиатуре (а сканер для системы и есть клавиатура) система **намеренно НЕ показывает экранную клавиатуру**. Это блокирует ручной ввод, когда штрихкод стёрт/не считывается.

### Что было сделано в этом блоке (хронологически)

**Шаг 1. Первая попытка — FAB + скрытое поле-донор.**
- Создан компонент `src/components/KeyboardFab.tsx` — рыжая плавающая кнопка в правом нижнем углу.
- Логика детекта сканера: ловим серию очень быстрых `keydown` (>=6 символов с интервалом <50мс) или `keydown` вне поля ввода. Если сработало — сохраняем в `localStorage` ключ `scanner_detected_at` (TTL 30 дней) и показываем FAB.
- При тапе по FAB пытались открыть системную клавиатуру через скрытый `<input>` + `focus()`.
- **Результат:** на реальном iPhone (Safari + Chrome) клавиатура НЕ открывалась — iOS игнорирует focus при подключённой BT-клавиатуре.

**Шаг 2. Вторая попытка — выезжающая панель ввода.**
- Добавили выезжающую снизу панель с собственным input-ом (`panelInputRef`).
- При тапе FAB — панель показывалась, но фокус на поле панели тоже не вызывал системную клавиатуру (та же причина — iOS).
- **Результат:** не работает по той же причине.

**Шаг 3. Принятое решение — собственная виртуальная клавиатура (вариант А).**
Раз iOS не даёт системную клавиатуру при подключённом BT-сканере — рисуем свою на HTML/CSS. От системной не зависит, работает гарантированно.

### Финальная реализация

**Файлы:**
- `src/components/KeyboardFab.tsx` — детектор сканера + FAB + кнопка скрыть «×».
- `src/components/VirtualKeyboard.tsx` — собственная клавиатура, появляется снизу при тапе FAB.

**Возможности `VirtualKeyboard`:**
- 3 раскладки: **АБВ** (ЙЦУКЕН) / **ABC** (QWERTY) / **123** (цифровая). Переключение одной кнопкой по кругу.
- Для полей `type="number"` или `inputMode="numeric"` сразу открывается раскладка **123**.
- Shift (одноразовый, сбрасывается после ввода символа).
- Backspace, пробел, кнопка **«Готово»** (диспатчит `Enter` keydown в активное поле + `change`-событие, затем закрывает клавиатуру).
- **Превью-строка сверху** — показывает текущее значение активного поля + кнопка «Очистить» + «×» закрыть.
- Фокус с целевого поля **не теряется** при тапах по клавишам (`onMouseDown` + `onTouchStart` с `preventDefault` на корневом контейнере).
- Запись значения в React-стейт через нативный сеттер `HTMLInputElement.prototype.value` + диспатч `input`-события — обновляет любой controlled-инпут как настоящий ввод.
- `scrollIntoView({ block: 'center' })` при открытии — поле не остаётся под клавиатурой.

**`KeyboardFab` — финальное состояние:**
- Срабатывает только на iOS (`isIOS()` через UA + `ontouchend`).
- Слушает `focusin` на документе → запоминает последний активный input/textarea в `lastFocusedRef`.
- Детектит сканер (быстрая серия keydown / keydown вне поля).
- Показывает FAB-кнопку с иконкой клавиатуры + микро-кнопку «×» (скрыть до перезагрузки страницы — флаг в `sessionStorage["keyboard_fab_hidden"]`).
- При тапе по FAB:
  - Если есть `lastFocusedRef` — открываем `VirtualKeyboard` с этим полем как target.
  - Если нет — показываем тост «Сначала тапни по полю ввода».
